# คู่มือการเปิดใช้งานระบบ Donate Screen

**GitHub Repository:** https://github.com/idragforsale/donate

## ภาพรวมระบบ
| หน้า | URL | คำอธิบาย |
|------|-----|----------|
| จอ TV | `/display?venue=1` | แสดงผลบนจอโปรเจกเตอร์/TV |
| หน้า Donate | `/donate` | ผู้ชมกรอกข้อมูลและโดเนท |
| Admin Panel | `/admin` | อนุมัติ/ปฏิเสธ ดูคิว และรายได้ |

## ส่วนที่ 1 — ติดตั้งครั้งแรก (First Time Setup)

### 1.1 ติดตั้ง Node.js
ดาวน์โหลดและติดตั้ง Node.js v18+ จาก https://nodejs.org

ตรวจสอบว่าติดตั้งแล้ว:
```bash
node -v    # ต้องได้ v18 ขึ้นไป
npm -v
```

### 1.2 ติดตั้ง MySQL
ดาวน์โหลด MySQL Community Server จาก https://dev.mysql.com/downloads/
หรือใช้ XAMPP (มี MySQL รวมมาด้วย)

### 1.3 Clone โปรเจกต์จาก GitHub
```bash
git clone https://github.com/idragforsale/donate.git donate-screen
cd donate-screen
```

### 1.4 ติดตั้ง Dependencies
```bash
# Backend
cd backend
npm install

# Frontend (เปิด Terminal ใหม่)
cd frontend/donate-screen-frontend
npm install
```

### 1.5 ตั้งค่า Environment
คัดลอก `.env.example` แล้วสร้างไฟล์ `.env` ใหม่:
```bash
cd backend
copy .env.example .env    # Windows
```

แก้ไขไฟล์ `backend/.env`:
```env
DB_HOST=localhost
DB_USER=root
DB_PASS=รหัสผ่าน MySQL ของคุณ
DB_NAME=venue_donation
PORT=9000
```

### 1.6 สร้างฐานข้อมูล
```bash
# อยู่ที่ root ของโปรเจกต์
mysql -u root -p < backend/schema.sql
```

> ระบุรหัสผ่าน MySQL เมื่อถูกถาม
> คำสั่งนี้จะสร้าง Database `venue_donation` + tables + packages เริ่มต้น 4 รายการ

## ส่วนที่ 2 — เปิดใช้งานทุกครั้ง

### Terminal 1 — เปิด Backend
```bash
cd donate-screen/backend
node server.js
```

เมื่อสำเร็จจะเห็น:
```
Server at http://0.0.0.0:9000
Queue initialized from DB
```

### Terminal 2 — เปิด Frontend
```bash
cd donate-screen/frontend/donate-screen-frontend
npm run dev
```

### เข้าใช้งาน
| หน้า | URL |
|------|-----|
| หน้า Donate | http://localhost:5173/donate |
| Admin Panel | http://localhost:5173/admin |
| จอ TV | http://localhost:5173/display?venue=1 |

> จอ TV → กด **F11** เพื่อ Full Screen

## ส่วนที่ 3 — อัปโหลดโค้ดขึ้น GitHub

ทุกครั้งที่แก้ไขโค้ดแล้วต้องการ save ขึ้น GitHub:

```bash
cd donate-screen

# ดูว่าไฟล์ไหนเปลี่ยนแปลง

# เพิ่มไฟล์ที่แก้ไขทั้งหมด
```


## ส่วนที่ 4 — ย้ายงานไปทำเครื่องอื่น

### วิธีที่ 1: ผ่าน GitHub (แนะนำ)

**เครื่องเก่า — Push งานที่ทำล่าสุด:**
```bash
cd donate-screen
git add .
git commit -m "save before transfer"
git push
```

**เครื่องใหม่ — ติดตั้งและดึงโค้ด:**
```bash
# 1. ติดตั้ง Node.js และ MySQL ก่อน (ดูส่วนที่ 1.1 - 1.2)

# 2. Clone โปรเจกต์
git clone https://github.com/idragforsale/donate.git donate-screen
cd donate-screen

# 3. ติดตั้ง dependencies
cd backend && npm install
cd ../frontend/donate-screen-frontend && npm install

# 4. สร้างไฟล์ .env ใหม่ (ไม่ได้ติดมากับ git เพราะมีรหัสผ่าน)
cd ../../backend
copy .env.example .env
# แก้ .env ให้ถูกต้องกับเครื่องใหม่

# 5. สร้าง Database
mysql -u root -p < backend/schema.sql

# 6. เปิดใช้งาน (ดูส่วนที่ 2)
```

> **หมายเหตุ:** ไฟล์ `.env` และโฟลเดอร์ `uploads/` (รูปภาพของผู้ใช้)
> **ไม่ได้**ติดไปกับ git ต้องทำใหม่หรือ copy เองแยกต่างหาก

### วิธีที่ 2: Copy โฟลเดอร์โดยตรง (USB / LAN)

Copy โฟลเดอร์ `donate-screen` ทั้งหมดไปยังเครื่องใหม่ **ยกเว้น** `node_modules`:

```
สิ่งที่ต้อง copy:
✅ backend/ (ยกเว้น node_modules/)
✅ frontend/ (ยกเว้น node_modules/)
✅ .gitignore
✅ คู่มือ*.md
✅ backend/uploads/ (ถ้าต้องการเก็บรูปเดิม)
❌ backend/node_modules/ (ใหญ่มาก, ไม่ต้อง copy)
❌ frontend/donate-screen-frontend/node_modules/
```

จากนั้นที่เครื่องใหม่:
```bash
cd donate-screen/backend
npm install   # ติดตั้ง dependencies ใหม่

cd ../frontend/donate-screen-frontend
npm install
```

## การตั้งค่า Auto-Approve

เข้า Admin Panel → แท็บ **⚙️ ตั้งค่า**

| โหมด | คำอธิบาย |
|------|----------|
| ปิด (ค่าเริ่มต้น) | Admin ต้อง Approve ทุก Donation ก่อนเข้าคิว |
| เปิด | Donation เข้าคิวอัตโนมัติทันที ไม่ต้องรอ Admin |

---

## การแก้ปัญหาเบื้องต้น

| ปัญหา | วิธีแก้ |
|-------|---------|
| แพ็กเกจไม่โหลด ("กำลังโหลด...") | ตรวจว่า Backend ทำงานอยู่ที่ port 9000 และ run schema.sql แล้ว |
| รูปภาพไม่แสดงบนจอ TV | Restart backend หลังจาก upload รูป |
| จอ TV ไม่อัปเดต real-time | เช็ค browser console — Socket.IO ต้องแสดง connected |
| Database connection error | ตรวจค่า DB_HOST/DB_USER/DB_PASS ใน `backend/.env` |
| git push แล้ว error | รัน `git pull` ก่อน แล้วค่อย push อีกครั้ง |
