# คู่มือการเปิดใช้งานระบบ Donate Screen

## ภาพรวมระบบ
ระบบ Donate Screen ประกอบด้วย 4 ส่วน:
| หน้า | URL | คำอธิบาย |
|------|-----|----------|
| จอ TV | `/display` | แสดงผลบนจอโปรเจกเตอร์/TV |
| หน้า Donate | `/donate` | ผู้ชมกรอกข้อมูลและโดเนท |
| Admin Panel | `/admin` | อนุมัติ/ปฏิเสธ ดูคิว และรายได้ |

---

## ขั้นตอนที่ 1 — ติดตั้ง MySQL และสร้างฐานข้อมูล

### 1.1 รันคำสั่งสร้าง Database และตาราง
เปิด MySQL (Command Prompt หรือ MySQL Workbench) แล้วรัน:

```bash
mysql -u root -p < backend/schema.sql
```

> ระบุรหัสผ่านเมื่อถูกถาม

คำสั่งนี้จะสร้าง:
- Database ชื่อ `venue_donation`
- ตาราง: `venues`, `packages`, `donations`, `queue_items`
- ข้อมูล packages เริ่มต้น 4 แพ็กเกจ

---

## ขั้นตอนที่ 2 — ตั้งค่า Environment (Backend)

ไฟล์ `backend/.env` ควรมีเนื้อหาตามนี้:

```env
DB_HOST=localhost
DB_USER=root
DB_PASS=รหัสผ่าน MySQL ของคุณ
DB_NAME=venue_donation
PORT=9000
```

> หากยังไม่มีไฟล์ `.env` ให้คัดลอกจาก `.env.example` แล้วแก้ไขค่าให้ถูกต้อง

---

## ขั้นตอนที่ 3 — เปิด Backend Server

```bash
cd backend
node server.js
```

หรือโหมด auto-restart (ต้องใช้ Node.js v18+):

```bash
cd backend
node --watch server.js
```

เมื่อ server ทำงานจะเห็น:
```
Server at http://0.0.0.0:9000
Queue initialized from DB
```

---

## ขั้นตอนที่ 4 — เปิด Frontend (Development)

เปิด Terminal ใหม่อีกหนึ่งหน้าต่าง:

```bash
cd frontend/donate-screen-frontend
npm install        # ครั้งแรกเท่านั้น
npm run dev
```

Frontend จะรันที่ `http://localhost:5173`

---

## ขั้นตอนที่ 5 — เข้าใช้งาน

| หน้า | URL |
|------|-----|
| หน้า Donate (ผู้ชม) | http://localhost:5173/donate |
| Admin Panel | http://localhost:5173/admin |
| จอ TV (Display) | http://localhost:5173/display |

### แนะนำการตั้งค่าหน้าจอ:
- เปิด **`/display`** บน TV หรือโปรเจกเตอร์ (Full Screen: กด F11)
- เปิด **`/admin`** บนคอมพิวเตอร์ของ Admin
- แชร์ลิงก์ **`/donate`** หรือ QR Code ให้ผู้ชมสแกน

---

## การตั้งค่า Auto-Approve

เข้า Admin Panel → แท็บ **⚙️ ตั้งค่า**

| โหมด | คำอธิบาย |
|------|----------|
| ปิด (ค่าเริ่มต้น) | Admin ต้อง Approve ทุก Donation ก่อนเข้าคิว |
| เปิด | Donation เข้าคิวอัตโนมัติทันที ไม่ต้องรอ Admin |

## โครงสร้างไฟล์โปรเจกต์

```

├── backend/
│   ├── server.js        ← Backend หลัก (Fastify + Socket.IO)
│   ├── db.js            ← MySQL connection pool

    ├── src/

```

## การแก้ปัญหาเบื้องต้น

### แพ็กเกจไม่โหลด ("กำลังโหลด...")
→ ตรวจสอบว่า Backend ทำงานอยู่ที่ port 9000
→ ตรวจสอบว่า run `schema.sql` แล้ว (มีข้อมูลใน packages table)

### รูปภาพที่อัปโหลดไม่แสดง
→ ตรวจสอบว่ามีโฟลเดอร์ `backend/uploads/` (สร้างอัตโนมัติเมื่อ server เริ่ม)
→ ตรวจสอบ permission ของโฟลเดอร์ uploads

### จอ TV ไม่อัปเดต real-time
→ ตรวจสอบว่า Socket.IO เชื่อมต่อได้ (ดูที่ browser console)
→ ตรวจสอบว่า frontend URL ชี้ไปที่ backend port 9000

### Database connection error
→ ตรวจสอบค่าใน `backend/.env` ให้ถูกต้อง
→ ตรวจสอบว่า MySQL service กำลังทำงานอยู่
